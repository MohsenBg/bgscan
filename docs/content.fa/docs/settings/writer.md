# تنظیمات نویسنده نتایج (Result Writer Settings)

> 💡 **نکته:** شما می‌توانید این تنظیمات را به جای ویرایش دستی فایل TOML، مستقیماً درون خود برنامه bgscan تغییر دهید.
>
> برای پیکربندی تعاملی این گزینه‌ها با استفاده از بخش مدیریت تنظیمات (TUI inspector)، از منوی اصلی به مسیر **Settings** ← **General Settings** (تب دوم) بروید.

فایل پیکربندی: `settings/writer_settings.toml`

این فایل فرآیند بافر کردن (ذخیره موقت) و فواصل زمانی انتقال و ثبت نتایج روی دیسک را کنترل می‌کند.

## مرجع سریع (Quick Reference)

| تنظیمات | مقدار پیش‌فرض | توضیحات |
|---------|---------|-------------|
| `merge_flush_interval` | `2000` | فاصله زمانی (به میلی‌ثانیه) برای ادغام نتایج تغییریافته در فایل اصلی نتایج |
| `chan_size` | `4096` | ظرفیت کانال داخلی مورد استفاده ورکرهای اسکنر برای ارسال نتایج اسکن آی‌پی به گوروتینِ (Goroutine) نویسنده |
| `batch_size` | `4096` | ظرفیت اولیه بافرِ داخل حافظه (In-memory) برای جمع‌آوری نتایج اسکن آی‌پی پیش از ذخیره روی دیسک |

---

## فواصل زمانی ادغام و تخلیه (Merge Flush Interval)

مشخص می‌کند که نتایج جزئی و خام اسکن در چه فواصل زمانی (به میلی‌ثانیه) روی دیسک ثبت نهایی (Commit) شوند. مقادیر کمتر نرخ خواندن/نوشتن (I/O) دیسک را افزایش اما میزان مصرف حافظه را کاهش می‌دهند. مقادیر بالاتر نرخ I/O دیسک را کاهش اما مصرف حافظه را افزایش می‌دهند.

```toml
merge_flush_interval = 2000

```

**مقادیر توصیه‌شده:**

* `1000-5000` برای مصارف معمول
* مقادیر کمتر برای سیستم‌هایی با حافظه ذخیره‌سازی پرسرعت (مثل SSDهای NVMe)
* مقادیر بالاتر برای سیستم‌هایی با پهنای باند I/O محدود

---

## ظرفیت کانال (Channel Size)

ظرفیت کانال ارتباطی داخلی را تعیین می‌کند که ورکرهای اسکنر از طریق آن، نتایج اسکن آی‌پی‌ها را به گوروتینِ نویسنده (Writer Goroutine) ارسال می‌کنند. مقادیر بالاتر به جلوگیری از ایجاد گلوگاه (Bottleneck) در عملکرد ورکرها هنگام پردازش سنگین دیسک کمک می‌کند.

```toml
chan_size = 4096

```

**مقادیر توصیه‌شده:**

* `1024-4096` برای نرخ پردازش متوسط
* `4096-16384` برای اسکن‌هایی با نرخ پردازش بالا (High-throughput)

---

## حجم هر دسته (Batch Size)

ظرفیت اولیه دسته‌ها در داخل حافظه (RAM) را مشخص می‌کند که برای جمع‌آوری نتایج اسکن آی‌پی پیش از تخلیه و ثبت آن‌ها روی دیسک کاربرد دارد. مقادیر بالاتر تعداد دفعات نوشتن روی دیسک را کاهش اما میزان مصرف حافظه رم را افزایش می‌دهند.

```toml
batch_size = 4096

```

**مقادیر توصیه‌شده:**

* `1024-4096` برای سیستم‌های دارای محدودیت حافظه رم
* `4096-16384` برای سیستم‌هایی با حافظه رم فراوان
