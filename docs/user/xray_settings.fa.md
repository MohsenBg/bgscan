# پیکربندی اسکنر Xray

فایل پیکربندی: `settings/xray_settings.toml`

این فایل ماژول تست اتصال Xray را کنترل می‌کند — شامل همزمانی worker‌ها، نوع تست، آستانه‌های سرعت، و استراتژی اختیاری پیش‌اسکن.

## هسته اصلی

### `timeout`

```toml
timeout = 6000
```

زمان مجاز بر حسب **میلی‌ثانیه** برای هر تست اتصال Xray. اگر هدف در این بازه پاسخ ندهد، تلاش به عنوان شکست‌خورده ثبت می‌شود.

---

### `workers`

```toml
workers = 32
```

تعداد goroutine‌های worker همزمان Xray.

| مقدار         | تعادل                                             |
| ------------- | ------------------------------------------------- |
| کم (`8`–`16`) | مصرف کمتر پهنای باند و CPU، توان عملیاتی پایین‌تر |
| متوسط (`32`)  | پیش‌فرض متعادل                                    |
| زیاد (`64`+)  | بیشترین توان عملیاتی، مصرف بیشتر پهنای باند و CPU |

## تست اتصال

### `connectivity_test_type`

```toml
connectivity_test_type = 3
```

نوع تست اتصالی که روی هر هدف انجام می‌شود را مشخص می‌کند.

| مقدار | حالت                | توضیح                                          |
| ----- | ------------------- | ---------------------------------------------- |
| `0`   | `ConnectivityOnly`  | بررسی پایه دسترس‌پذیری — بدون اندازه‌گیری سرعت |
| `1`   | `DownloadSpeedOnly` | فقط سرعت دانلود اندازه‌گیری می‌شود             |
| `2`   | `UploadSpeedOnly`   | فقط سرعت آپلود اندازه‌گیری می‌شود              |
| `3`   | `Both`              | هر دو تست دانلود و آپلود انجام می‌شوند         |

---

## آستانه‌های سرعت

این تنظیمات **حداقل سرعت قابل قبول** برای قبول شدن یک هدف را مشخص می‌کنند. هدف‌هایی که زیر این آستانه هستند حذف می‌شوند.

### `download_speed`

```toml
download_speed = 100
```

حداقل سرعت دانلود مورد نیاز بر حسب **Kbps**.

> فقط زمانی اعمال می‌شود که `connectivity_test_type` برابر `1` (DownloadSpeedOnly) یا `3` (Both) باشد.

---

### `upload_speed`

```toml
upload_speed = 50
```

حداقل سرعت آپلود مورد نیاز بر حسب **Kbps**.

> فقط زمانی اعمال می‌شود که `connectivity_test_type` برابر `2` (UploadSpeedOnly) یا `3` (Both) باشد.

---

## خروجی

### `prefix_output`

```toml
prefix_output = "xray_"
```

پیشوند اضافه‌شده به فایل‌های خروجی این اسکنر (مثلاً `xray_results.txt`). در زمان اجرای چند نوع اسکنر به‌صورت موازی از تداخل نام فایل‌ها جلوگیری می‌کند.

---

## پیش‌اسکن

### `pre_scan_type`

```toml
pre_scan_type = "tcp"
```

یک بررسی سبک و اختیاری است که **قبل** از اجرای تست کامل Xray انجام می‌شود. هدف‌هایی که در پیش‌اسکن رد می‌شوند کاملاً نادیده گرفته می‌شوند و در زمان و منابع صرفه‌جویی می‌شود.

| مقدار    | رفتار                                                        |
| -------- | ------------------------------------------------------------ |
| `"none"` | بدون پیش‌اسکن — هدف‌ها مستقیماً به Xray می‌روند              |
| `"icmp"` | ICMP ping به IP هدف (از تنظیمات `icmp` استفاده می‌کند)       |
| `"tcp"`  | اتصال TCP به پورت هدف (از تنظیمات `tcp` استفاده می‌کند)      |
| `"http"` | درخواست HTTP/HTTPS به هدف (از تنظیمات `http` استفاده می‌کند) |

> هر نوع پیش‌اسکن پارامترهای خود (timeout، پورت و ...) را از بخش تنظیمات مربوطه می‌خواند.

---

## مثال کامل

```toml
# ── هسته اصلی ─────────────────────────────────────────────────
timeout = 6000   # timeout 6 ثانیه برای هر تست
workers = 32     # 32 worker همزمان

# ── تست اتصال ─────────────────────────────────────────────────
connectivity_test_type = 3   # هر دو دانلود و آپلود تست شوند

# ── آستانه‌های سرعت ───────────────────────────────────────────
download_speed = 100   # حداقل ۱۰۰ Kbps دانلود
upload_speed   = 50    # حداقل ۵۰ Kbps آپلود

# ── خروجی ─────────────────────────────────────────────────────
prefix_output = "xray_"

# ── پیش‌اسکن ──────────────────────────────────────────────────
pre_scan_type = "tcp"   # بررسی TCP قبل از اجرای Xray
```
