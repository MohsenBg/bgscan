# تنظیمات TCP (TCP Settings)

> 💡 **نکته:** شما می‌توانید این تنظیمات را به جای ویرایش دستی فایل TOML، مستقیماً درون خود برنامه bgscan تغییر دهید.
>
> برای پیکربندی تعاملی این گزینه‌ها با استفاده از بخش مدیریت تنظیمات (TUI inspector)، از منوی اصلی به مسیر **Settings** ← **TCP Settings** بروید.

فایل پیکربندی: `settings/tcp_settings.toml`

این فایل پارامترهای مربوط به اسکن پورت TCP از جمله پورت هدف، مهلت زمانی (Timeout)، تعداد ورکرها، دفعات تلاش مجدد و پیشوند خروجی را کنترل می‌کند.

## پورت (Port)

```toml
port = 80

```

پورت TCP هدف که روی هر آدرس آی‌پی بررسی می‌شود. اسکنر تلاش می‌کند یک اتصال TCP با این پورت برقرار کند.
نمونه‌های رایج: ۸۰ (HTTP)، ۴۴۳ (HTTPS)، ۲۲ (SSH).

**مقادیر توصیه‌شده:** پورت‌های رایج مربوط به سرویسی که در حال اسکن آن هستید.

---

## مهلت زمانی (Timeout)

```toml
timeout = 3000

```

حداکثر زمان انتظار (به میلی‌ثانیه) برای برقراری اتصال TCP. اهدافی که در این بازه پاسخ ندهند، غیرقابل دسترس در نظر گرفته می‌شوند.

**مقادیر توصیه‌شده:**

* `1000-3000` برای شبکه‌های محلی
* `5000-10000` برای شبکه‌های دور یا ناپایدار

---

## ورکرها (Workers)

```toml
workers = 200

```

تعداد ورکرهای هم‌زمان که برای برقراری اتصال TCP تلاش می‌کنند. مقادیر بالاتر سرعت اسکن را افزایش می‌دهند، اما مصرف پردازنده (CPU) و پهنای باند شبکه را بالا می‌برند.

**مقادیر توصیه‌شده:**

* `50-100` برای سیستم‌هایی با منابع محدود
* `200-500` برای مصارف معمول و عادی
* `500-1000` برای اسکن با کارایی و سرعت بالا

---

## پیشوند خروجی (Prefix Output)

```toml
prefix_output = "tcp_"

```

پیشوند اعمال‌شده روی نام تمامی فایل‌های خروجی که توسط این ماژول تولید می‌شوند. این ویژگی برای جداسازی و تشخیص نتایج هنگامی که چندین اسکن را به صورت هم‌زمان اجرا می‌کنید، بسیار مفید است.

---

## دفعات تلاش مجدد (Retries)

```toml
tries = 1

```

حداکثر تعداد دفعات تلاش مجدد برای هر هدف جهت کاهش خطاهای احتمالی در اتصال. مقادیر بالاتر قابلیت اطمینان اسکن را بهبود می‌بخشند اما زمان کل اسکن را افزایش می‌دهند.

**مقادیر توصیه‌شده:**

* `1` بدون تلاش مجدد (سریع‌ترین حالت)
* `2-3` برای اکثر شبکه‌ها
* `4-10` برای شبکه‌های ناپایدار
