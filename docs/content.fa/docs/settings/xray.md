# تنظیمات Xray (Xray Settings)

> 💡 **نکته:** شما می‌توانید این تنظیمات را به جای ویرایش دستی فایل TOML، مستقیماً درون خود برنامه bgscan تغییر دهید.
>
> برای پیکربندی تعاملی این گزینه‌ها با استفاده از بخش مدیریت تنظیمات (TUI inspector)، از منوی اصلی به مسیر **Settings** ← **Xray Settings** بروید.

فایل پیکربندی: `settings/xray_settings.toml`

این فایل ماژول اعتبارسنجی خروجی‌های Xray را کنترل می‌کند که وظیفه تست اتصال و پهنای باند سرورهای راه دور را بر عهده دارد.

## مرجع سریع (Quick Reference)

| تنظیمات | مقدار پیش‌فرض | توضیحات |
|---------|---------|-------------|
| `timeout` | `6000` | حداکثر زمان انتظار (به میلی‌ثانیه) برای تکمیل تست اتصال Xray. |
| `workers` | `32` | تعداد ورکرهای هم‌زمان برای اجرای تست‌های اتصال Xray. |
| `connectivity_test_type` | `3` | نوع تست اتصال: 0 = فقط اتصال اولیه، 1 = فقط دانلود، 2 = فقط آپلود، 3 = هر دو تست دانلود و آپلود. |
| `download_speed` | `100` | حجم داده هدف برای تست دانلود (به کیلوبایت)؛ در صورت گنجانده شدن دانلود در نوع تست، استفاده می‌شود. |
| `upload_speed` | `50` | حجم داده هدف برای تست آپلود (به کیلوبایت)؛ در صورت گنجانده شدن آپلود در نوع تست، استفاده می‌شود. |
| `prefix_output` | `"xray_"` | پیشوند اعمال‌شده روی نام تمامی فایل‌های خروجی تولید شده توسط این ماژول. |
| `pre_scan_type` | `"tcp"` | تست اتصال اولیه (Pre-scan) پیش از اجرای Xray: گزینه‌های `none`، `icmp`، `tcp` یا `http`. |

---

## مهلت زمانی (Timeout)

```toml
timeout = 6000

```

حداکثر زمان انتظار (به میلی‌ثانیه) برای به پایان رسیدن تست اتصال Xray. تست‌هایی که در این بازه پاسخ ندهند، ناموفق (Failed) در نظر گرفته می‌شوند.

**مقادیر توصیه‌شده:**

* `3000-6000` برای شبکه‌های پایدار و مطمئن
* `6000-12000` برای شبکه‌های ناپایدار یا دور

---

## ورکرها (Workers)

```toml
workers = 32

```

تعداد ورکرهای هم‌زمان که تست‌های اتصال Xray را انجام می‌دهند. مقادیر بالاتر سرعت و نرخ تست را افزایش می‌دهند اما به قیمت مصرف پهنای باند و منابع پردازنده (CPU) بیشتر خواهد بود.

**مقادیر توصیه‌شده:**

* `8-16` برای سیستم‌هایی با منابع محدود
* `16-32` برای مصارف معمول و عادی
* `32-64` برای تست با کارایی و سرعت بالا

---

## نوع تست اتصال (Connectivity Test Type)

```toml
connectivity_test_type = 3

```

نوع تست اتصالی که باید روی اهداف اجرا شود.

**مقادیر پشتیبانی‌شده:**

* `0` = فقط اتصال اولیه (Connectivity only) ← بررسی اینکه آیا سرور در دسترس است یا خیر (اتصال پایه)
* `1` = فقط سرعت دانلود (Download speed only) ← سنجش و اندازه‌گیری کارایی دانلود
* `2` = فقط سرعت آپلود (Upload speed only) ← سنجش و اندازه‌گیری کارایی آپلود
* `3` = هر دو حالت (Both) ← اجرای هم‌زمان تست سرعت دانلود و آپلود

**مقدار پیش‌فرض:** `3` (هر دو حالت دانلود و آپلود) — برای انجام یک ارزیابی جامع و کامل توصیه می‌شود.

---

## سرعت دانلود (Download Speed)

```toml
download_speed = 100

```

حجم داده هدف برای تست دانلود (به کیلوبایت). تست تلاش می‌کند این مقدار داده را در طول مهلت زمانی تعیین‌شده دانلود کند.
تنها زمانی اعمال می‌شود که `connectivity_test_type` شامل دانلود باشد (یعنی مقدار `1` یا `3`).

**مقادیر توصیه‌شده:**

* `50-200` برای تست پهنای باند متوسط
* `200-1000` برای تست پهنای باند بالا و پرسرعت

---

## سرعت آپلود (Upload Speed)

```toml
upload_speed = 50

```

حجم داده هدف برای تست آپلود (به کیلوبایت). تست تلاش می‌کند این مقدار داده را در طول مهلت زمانی تعیین‌شده آپلود کند.
تنها زمانی اعمال می‌شود که `connectivity_test_type` شامل آپلود باشد (یعنی مقدار `2` یا `3`).

**مقادیر توصیه‌شده:**

* `25-100` برای تست پهنای باند متوسط
* `100-500` برای تست پهنای باند بالا و پرسرعت

---

## پیشوند خروجی (Prefix Output)

```toml
prefix_output = "xray_"

```

پیشوند اعمال‌شده روی نام تمامی فایل‌های خروجی که توسط این ماژول تولید می‌شوند. این ویژگی برای جداسازی و تشخیص نتایج هنگامی که چندین اسکن را به صورت هم‌زمان اجرا می‌کنید، بسیار مفید است.

---

## نوع اسکن اولیه (Pre-scan Type)

```toml
pre_scan_type = "tcp"

```

نوع تست اتصال اولیه‌ای که قرار است قبل از درگیر کردن و اجرای فرآیند Xray انجام شود.
گزینه‌ها:

* `none` ← نادیده گرفتن اسکن اولیه و اجرای مستقیم Xray
* `icmp` ← تست پینگ ICMP به آی‌پ‌ی هدف (از تنظیمات ICMP استفاده می‌کند)
* `tcp` ← تست اتصال TCP به پورت هدف (از تنظیمات TCP استفاده می‌کند)
* `http` ← تست درخواست HTTP/HTTPS به هدف (از تنظیمات HTTP استفاده می‌کند)

**مقدار پیش‌فرض:** `"tcp"` — توصیه می‌شود تا پیش از انجام تست پهنای باند، از اتصال پایه شبکه اطمینان حاصل شود.
