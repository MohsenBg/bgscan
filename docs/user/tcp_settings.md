## TCP Settings

Configuration file: `settings/tcp_settings.toml`

These settings control the behavior of the TCP scanner, including the target port, connection timeout, concurrency level, retry policy, and output file naming.

---

### `port`

```toml
port = 80
```

TCP port that will be tested on each target IP address.

The scanner attempts to establish a TCP connection to this port. A successful connection indicates that the target service is reachable and accepting connections.

| Value | Common Service |
| ----- | -------------- |
| `80`  | HTTP           |
| `443` | HTTPS          |
| `22`  | SSH            |
| `21`  | FTP            |
| `25`  | SMTP           |

> The selected port should match the service you intend to verify.

---

### `timeout`

```toml
timeout = 3000
```

Maximum time to wait for a TCP connection attempt to complete, measured in **milliseconds**.

If the connection is not established within this period, the attempt is considered failed.

| Value  | Behavior                           |
| ------ | ---------------------------------- |
| `1000` | Wait up to 1 second                |
| `3000` | Wait up to 3 seconds (default)     |
| `5000` | Suitable for high-latency networks |

> Increasing this value may improve detection rates on slower networks but will increase overall scan duration.

---

### `workers`

```toml
workers = 200
```

Number of concurrent worker goroutines performing TCP connection attempts.

Higher values increase scanning throughput but also increase CPU usage, memory consumption, and the number of simultaneous network connections.

| Value   | Behavior                                |
| ------- | --------------------------------------- |
| `50`    | Low resource usage                      |
| `200`   | Balanced performance (default)          |
| `1000+` | High-speed scanning on powerful systems |

> Excessively high values may exhaust local resources or trigger connection limits on some operating systems.

---

### `tries`

```toml
tries = 1
```

Number of connection attempts performed for each target.

If a connection attempt fails, the scanner will retry until the configured limit is reached.

| Value | Behavior                                  |
| ----- | ----------------------------------------- |
| `1`   | Single attempt (default)                  |
| `2`   | One retry if the first attempt fails      |
| `3+`  | Improved reliability on unstable networks |

> Higher values can reduce false negatives but increase total scan time.

---

### `prefix_output`

```toml
prefix_output = "tcp_"
```

Prefix used when naming files generated by this scanner.

This is useful when multiple scanner types are used within the same project and their results need to be separated.

| Value       | Example Output       |
| ----------- | -------------------- |
| `"tcp_"`    | `tcp_results.txt`    |
| `"web_"`    | `web_results.txt`    |
| `"custom_"` | `custom_results.txt` |

> This setting only affects output file names and does not change scanning behavior.
