# Xray Scanner Configuration

Configuration file: `settings/xray_settings.toml`

This file controls the Xray connectivity testing module — including worker concurrency, test types, speed thresholds, and the optional pre-scan strategy.

## Core

### `timeout`

```toml
timeout = 6000
```

Time in **milliseconds** allowed for each Xray connectivity test. If the target does not respond within this window, the attempt is marked as failed.

---

### `workers`

```toml
workers = 32
```

Number of concurrent Xray worker goroutines.

| Value          | Trade-off                                            |
| -------------- | ---------------------------------------------------- |
| Low (`8`–`16`) | Reduced bandwidth and CPU usage, lower throughput    |
| Medium (`32`)  | Balanced default                                     |
| High (`64`+)   | Maximum throughput, increased bandwidth and CPU load |

---

## Connectivity Test

### `connectivity_test_type`

```toml
connectivity_test_type = 3
```

Defines what type of connectivity test is performed on each target.

| Value | Mode                | Description                                     |
| ----- | ------------------- | ----------------------------------------------- |
| `0`   | `ConnectivityOnly`  | Basic reachability check — no speed measurement |
| `1`   | `DownloadSpeedOnly` | Measures download performance only              |
| `2`   | `UploadSpeedOnly`   | Measures upload performance only                |
| `3`   | `Both`              | Performs both download and upload speed tests   |

---

## Speed Thresholds

These settings define the **minimum acceptable speed** for a target to pass the test. Targets that fall below the threshold are discarded.

### `download_speed`

```toml
download_speed = 100
```

Minimum required download speed in **Kbps**.

> Only applies when `connectivity_test_type` is `1` (DownloadSpeedOnly) or `3` (Both).

---

### `upload_speed`

```toml
upload_speed = 50
```

Minimum required upload speed in **Kbps**.

> Only applies when `connectivity_test_type` is `2` (UploadSpeedOnly) or `3` (Both).

---

## Output

### `prefix_output`

```toml
prefix_output = "xray_"
```

Prefix added to output files generated by this scanner (e.g., `xray_results.txt`). Useful when running multiple scanner types in parallel to avoid file name collisions.

---

## Pre-Scan

### `pre_scan_type`

```toml
pre_scan_type = "tcp"
```

An optional lightweight connectivity check performed **before** running the full Xray test. Targets that fail the pre-scan are skipped entirely, saving time and resources.

| Value    | Behavior                                                |
| -------- | ------------------------------------------------------- |
| `"none"` | No pre-scan — targets go directly to Xray               |
| `"icmp"` | ICMP ping to the target IP (uses `icmp` settings)       |
| `"tcp"`  | TCP connection to the target port (uses `tcp` settings) |
| `"http"` | HTTP/HTTPS request to the target (uses `http` settings) |

> Each pre-scan type reads its parameters (timeout, port, etc.) from its respective settings section.

---

## Full Example

```toml
# ── Core ──────────────────────────────────────────────────────
timeout = 6000   # 6-second per-test timeout
workers = 32     # 32 concurrent workers

# ── Connectivity Test ─────────────────────────────────────────
connectivity_test_type = 3   # Test both download and upload

# ── Speed Thresholds ──────────────────────────────────────────
download_speed = 100   # Minimum 100 Kbps download
upload_speed   = 50    # Minimum 50 Kbps upload

# ── Output ────────────────────────────────────────────────────
prefix_output = "xray_"

# ── Pre-Scan ──────────────────────────────────────────────────
pre_scan_type = "tcp"   # TCP check before running Xray
```
