# پیکربندی Result Writer

فایل پیکربندی: `settings/result_writer.toml`

این فایل نحوه جمع‌آوری نتایج اسکن از worker goroutine‌ها، انباشت آن‌ها در حافظه، و نوشتن به دیسک را کنترل می‌کند.

---

## تنظیمات

### `merge_flush_interval`

```toml
merge_flush_interval = 2000
```

فاصله زمانی بر حسب **میلی‌ثانیه** که در آن نتایج delta به فایل نتیجه اصلی ادغام می‌شوند.

| مقدار  | رفتار                              |
| ------ | ---------------------------------- |
| `1000` | هر ۱ ثانیه — تأخیر کمتر، I/O بیشتر |
| `2000` | هر ۲ ثانیه (پیش‌فرض)               |
| `5000` | هر ۵ ثانیه — I/O کمتر، تأخیر بیشتر |

---

### `chan_size`

```toml
chan_size = 4096
```

ظرفیت کانال داخلی که worker‌های اسکنر از آن برای ارسال نتایج `IPScanResult` به goroutine نویسنده استفاده می‌کنند.

بافر بزرگ‌تر احتمال مسدود شدن worker‌ها را در زمان flush شدن نویسنده به دیسک کاهش می‌دهد.

| مقدار             | تعادل                                               |
| ----------------- | --------------------------------------------------- |
| کم (`512`–`1024`) | حافظه کمتر، احتمال block شدن worker‌ها در بار سنگین |
| متوسط (`4096`)    | پیش‌فرض متعادل                                      |
| زیاد (`8192`+)    | کاهش stall، مصرف حافظه بیشتر                        |

---

### `batch_size`

```toml
batch_size = 4096
```

ظرفیت اولیه batch درون‌حافظه‌ای که برای انباشت نتایج `IPScanResult` قبل از نوشتن به دیسک استفاده می‌شود.

این یک راهنمای پیش‌تخصیص است — اگر نیاز باشد batch می‌تواند از این مقدار فراتر رود، اما شروع با اندازه مناسب از reallocation‌های مکرر حافظه در اسکن‌های پرسرعت جلوگیری می‌کند.

| مقدار             | تعادل                               |
| ----------------- | ----------------------------------- |
| کم (`512`–`1024`) | مصرف حافظه پایین‌تر، flush بیشتر    |
| متوسط (`4096`)    | پیش‌فرض متعادل                      |
| زیاد (`8192`+)    | flush کمتر، مصرف حافظه بیشتر در اوج |

---

## مثال کامل

```toml
# ── Result Writer ─────────────────────────────────────────────
merge_flush_interval = 2000   # هر ۲ ثانیه نتایج delta ادغام شوند
chan_size             = 4096   # بافر کانال بین worker‌ها و نویسنده
batch_size           = 4096   # ظرفیت batch درون‌حافظه‌ای قبل از flush به دیسک
```
