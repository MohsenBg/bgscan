# پیکربندی اسکنر HTTP

فایل پیکربندی: `settings/http_settings.toml`

این فایل رفتار تمام ماژول‌های اسکن مبتنی بر HTTP و HTTPS را کنترل می‌کند و امکان تنظیم پارامترهای امنیت انتقال، زمان‌بندی درخواست‌ها و سطح همزمانی را فراهم می‌کند.

## `[http]` — اسکنر اتصال

این ماژول درخواست‌های مستقیم HTTP/HTTPS را به هاست‌های هدف ارسال می‌کند تا وضعیت دسترسی شبکه، هدرهای سرور وب و هندشیک‌های رمزنگاری بررسی شوند.

---

## هسته اصلی

### `host`

```toml
host = "example.com"
```

هاست هدف یا مسیر URL مورد استفاده برای درخواست HTTP.

این مقدار می‌تواند یک دامنه ساده یا همراه با مسیر باشد.

| مقدار              | توضیح            |
| ------------------ | ---------------- |
| `example.com`      | فقط دامنه        |
| `example.com/path` | دامنه همراه مسیر |

> اگر مسیر مشخص شود، در URL درخواست استفاده خواهد شد، اما هدر `Host` همیشه فقط شامل دامنه است.

---

### `port`

```toml
port = 443
```

پورت هدف برای اتصال HTTP/HTTPS.

| مقدار | توضیح           |
| ----- | --------------- |
| `80`  | HTTP استاندارد  |
| `443` | HTTPS استاندارد |

---

### `protocol`

```toml
protocol = "https"
```

پروتکل مورد استفاده برای اسکن.

| مقدار     | توضیح                |
| --------- | -------------------- |
| `"http"`  | ارتباط بدون رمزنگاری |
| `"https"` | ارتباط رمزنگاری‌شده  |

---

## زمان‌بندی و همزمانی

### `timeout`

```toml
timeout = 4000
```

زمان انتظار برای هر درخواست HTTP بر حسب میلی‌ثانیه.

اگر سرور در این بازه پاسخ ندهد، درخواست ناموفق در نظر گرفته می‌شود.

---

### `workers`

```toml
workers = 50
```

تعداد workerهای همزمان برای اجرای درخواست‌ها.

| بازه پیشنهادی | توضیح                       |
| ------------- | --------------------------- |
| `10 – 150`    | تعادل بین سرعت و مصرف منابع |

> افزایش مقدار باعث افزایش سرعت ولی همچنین افزایش مصرف CPU و شبکه می‌شود.

---

## تنظیمات TLS

### `tls_validation`

```toml
tls_validation = true
```

فعال یا غیرفعال کردن بررسی اعتبار گواهی TLS.

| مقدار   | توضیح                                 |
| ------- | ------------------------------------- |
| `true`  | فقط گواهی معتبر پذیرفته می‌شود        |
| `false` | گواهی‌های نامعتبر نیز پذیرفته می‌شوند |

---

### `min_tls_version`

```toml
min_tls_version = "tls1.1"
```

حداقل نسخه TLS مجاز.

| مقدار      |
| ---------- |
| `"tls1.0"` |
| `"tls1.1"` |
| `"tls1.2"` |
| `"tls1.3"` |

---

### `max_tls_version`

```toml
max_tls_version = "tls1.3"
```

حداکثر نسخه TLS مجاز برای هندشیک.

---

### `server_name`

```toml
server_name = ""
```

SNI (Server Name Indication) برای اتصال TLS.

| مقدار      | توضیح                  |
| ---------- | ---------------------- |
| خالی       | استفاده خودکار از host |
| مقدار دستی | تعیین نام سرور TLS     |

---

## خروجی

### `prefix_output`

```toml
prefix_output = "http_"
```

پیشوند نام فایل‌های خروجی تولیدشده توسط اسکنر.

| مثال      | خروجی              |
| --------- | ------------------ |
| `"http_"` | `http_results.txt` |

> برای تفکیک خروجی چند اسکنر مختلف در یک پروژه استفاده می‌شود.

---

## مثال کامل

```toml
[http]
host            = "example.com"
port            = 443
protocol        = "https"
timeout         = 4000
workers         = 50
tls_validation  = true
min_tls_version = "tls1.1"
max_tls_version = "tls1.3"
server_name     = ""
prefix_output   = "http_"
```
