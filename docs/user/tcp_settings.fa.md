## تنظیمات TCP

این تنظیمات رفتار اسکنر TCP را کنترل می‌کنندg از پورت هدف و زمان انتظار اتصال گرفته تا تعداد تلاش‌ها، میزان همزمانی و نام‌گذاری فایل‌های خروجی.

---

### `port`

```toml
port = 80
```

پورتی که روی هر IP هدف بررسی می‌شود.

اسکنر تلاش می‌کند یک اتصال TCP به این پورت برقرار کند. برقراری موفق اتصال نشان می‌دهد که سرویس موردنظر در دسترس بوده و درخواست‌های ورودی را می‌پذیرد.

| مقدار | سرویس متداول |
| ----- | ------------ |
| `80`  | HTTP         |
| `443` | HTTPS        |
| `22`  | SSH          |
| `21`  | FTP          |
| `25`  | SMTP         |

> پورت انتخاب‌شده باید با سرویسی که قصد بررسی آن را دارید مطابقت داشته باشد.

---

### `timeout`

```toml
timeout = 3000
```

حداکثر مدت زمان انتظار برای برقراری اتصال TCP، بر حسب **میلی‌ثانیه**.

اگر اتصال در این بازه زمانی برقرار نشود، تلاش ناموفق در نظر گرفته می‌شود.

| مقدار  | رفتار                        |
| ------ | ---------------------------- |
| `1000` | انتظار تا ۱ ثانیه            |
| `3000` | انتظار تا ۳ ثانیه (پیش‌فرض)  |
| `5000` | مناسب شبکه‌های با تأخیر بالا |

> افزایش این مقدار می‌تواند شناسایی سرویس‌های کندتر را بهبود دهد، اما زمان کلی اسکن را نیز افزایش می‌دهد.

---

### `workers`

```toml
workers = 200
```

تعداد Workerهای همزمان که وظیفه برقراری اتصال TCP را بر عهده دارند.

مقادیر بالاتر باعث افزایش سرعت اسکن می‌شوند، اما مصرف CPU، حافظه و تعداد اتصال‌های همزمان شبکه را نیز افزایش می‌دهند.

| مقدار   | رفتار                                       |
| ------- | ------------------------------------------- |
| `50`    | مصرف منابع کم                               |
| `200`   | تعادل مناسب بین سرعت و مصرف منابع (پیش‌فرض) |
| `1000+` | مناسب اسکن‌های پرسرعت روی سیستم‌های قدرتمند |

> مقادیر بسیار بالا ممکن است باعث محدودیت سیستم‌عامل یا افزایش فشار روی شبکه شود.

---

### `tries`

```toml
tries = 1
```

تعداد دفعاتی که برای هر هدف تلاش اتصال انجام می‌شود.

در صورت شکست یک تلاش، اسکنر تا سقف تعیین‌شده دوباره تلاش خواهد کرد.

| مقدار | رفتار                              |
| ----- | ---------------------------------- |
| `1`   | فقط یک تلاش (پیش‌فرض)              |
| `2`   | یک تلاش مجدد در صورت شکست          |
| `3+`  | پایداری بیشتر در شبکه‌های ناپایدار |

> افزایش تعداد تلاش‌ها می‌تواند احتمال از دست رفتن اهداف سالم را کاهش دهد، اما زمان اسکن را نیز افزایش می‌دهد.

---

### `prefix_output`

```toml
prefix_output = "tcp_"
```

پیشوندی که به نام فایل‌های خروجی تولیدشده توسط این اسکنر اضافه می‌شود.

این گزینه برای تفکیک نتایج اسکنرهای مختلف در یک پروژه مفید است.

| مقدار       | نمونه خروجی          |
| ----------- | -------------------- |
| `"tcp_"`    | `tcp_results.csv`    |
| `"web_"`    | `web_results.csv`    |
| `"custom_"` | `custom_results.csv` |

> این تنظیم فقط روی نام فایل‌های خروجی تأثیر می‌گذارد و رفتار اسکن را تغییر نمی‌دهد.
